1. FIX INCONSISTENCY: log the transition (do this first)
---------------------------------------------------------
provide_delivery_address sets status + commits but does NOT write a PrizeDeliveryLog,
even though the create flow does and create_prize_delivery_log has an `actor` param for it.
The audit trail has a hole at the first transition.

Add one helper in services/prize_delivery_service.py and route all status changes through it:

    def transition(prize_delivery, new_status, actor=None, note=None) -> bool:
        if not can_transition(prize_delivery, new_status):
            return False
        log = create_prize_delivery_log(
            prize_delivery, from_status=prize_delivery.status,
            to_status=new_status, actor=actor, note=note,
        )
        prize_delivery.status = new_status
        db.session.add(log)
        return True

Then update provide_delivery_address to call transition() instead of setting .status directly.


2. BUILD REMAINING TRANSITION ROUTES
------------------------------------
Each route pattern: authz -> validate -> transition() -> commit -> notify.

- PENDING_SENDER_ADDRESS -> WAITING_FOR_SHIPMENT
    Creator confirms pickup address. Form exists: forms/pickup_address_form.py. Authz = creator.
- WAITING_FOR_SHIPMENT -> PRIZE_SENT
    Creator marks shipped (optionally set/confirm tracking). Authz = creator.
- PRIZE_SENT -> PRIZE_DELIVERED | DELIVERY_FAILED
    Mark delivered / failed.
- PRIZE_DELIVERED -> PRIZE_ACCEPTED | CONTESTED
    Winner accepts or contests. Authz = winner.
- CONTESTED -> PRIZE_ACCEPTED | PRIZE_REJECTED
    Dispute resolution (likely admin).
